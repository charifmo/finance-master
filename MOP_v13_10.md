## §1 — Contexte et objectif

### Problème résolu
Le chat_history gère la mémoire de session (8 derniers échanges) mais les préférences et règles exprimées par l'utilisateur disparaissent à la session suivante.

### Solution déployée
Un nouvel outil `memory_writer` est ajouté à l'Agent CFO. Quand l'utilisateur dit explicitement "retiens que…", l'agent :
1. Appelle l'outil `memory_writer`
2. Celui-ci POST au sous-workflow `CFO Memory Ingest` (webhook interne n8n)
3. Le sous-workflow vectorise le texte via Gemini (`gemini-embedding-001`) et l'insère dans `finance_vectors`
4. À la session suivante, `finance_rag` retrouvera l'information par recherche sémantique

## §4 — Points d'attention critiques

### Dimension des vecteurs (IMPORTANT)
`gemini-embedding-001` produit des vecteurs de **768 dimensions**.  
Si `finance_vectors` a été créée avec OpenAI (1536 dimensions), les vecteurs seront **incompatibles**.

Le script détecte automatiquement la dimension. Si incompatibilité :
```sql
-- Option A : Recréer proprement (perte des anciens vecteurs)
TRUNCATE finance_vectors;

-- Option B : Renommer et créer une nouvelle table (conserver l'ancien)
ALTER TABLE finance_vectors RENAME TO finance_vectors_openai;
-- n8n recréera finance_vectors avec la bonne dimension au premier insert
```

### Aucune modification de la logique budget
Le patch v13.10 n'altère pas :
- `budget_engine` (v6, v13.9)
- `committer`
- `finance_rag` (retrieval)
- Le routing Telegram/Webhook
- La logique de redistribution proportionnelle (v13.9)

### Le sous-workflow est indépendant
Si le sous-workflow `CFO Memory Ingest` rencontre une erreur, le Super-Agent CFO continue de fonctionner normalement. L'outil `memory_writer` retournera une erreur explicite que l'agent communiquera à l'utilisateur. Zéro impact sur les autres fonctionnalités.

## §6 — Plan de rollback

### Rollback automatique (workflow principal)
```bash
# Le backup a été créé automatiquement par install_v13_10.sh
bash install_v13_10.sh --rollback backup_super_agent_YYYYMMDD_HHMMSS.json
```
→ Restaure le workflow principal à son état pré-déploiement en 10 secondes.

### Rollback manuel du sous-workflow
1. n8n UI → Workflows → CFO Memory Ingest → Désactiver
2. Supprimer le workflow si nécessaire

### Rollback DB (si vecteurs corrompus)
```sql
-- Supprimer uniquement les entrées v13.10 (par source)
DELETE FROM finance_vectors
WHERE metadata->>'source' = 'cfo_agent_memory_writer';
```

### Vérification post-rollback
Envoyer une question au chat CFO (diagnostic, pas de mémorisation) et vérifier que la réponse est normale. Le workflow v13.9 doit fonctionner identiquement.

## §7 — Architecture de la solution

```
Utilisateur → "Retiens que je préfère les Airbnb"
     ↓
AI Agent (Gemini 2.5 Pro)
  [détecte demande explicite de mémorisation]
     ↓
Tool: Memory Writer (toolCode)
  POST https://n8n.beau.ink/webhook/finance-memory-ingest
  Body: { texte, categorie, session_id }
     ↓
Sous-workflow CFO Memory Ingest
  ├── Validate & Prepare (Code) — enrichit le texte avec [CFO-MEMORY | CAT | DATE]
  ├── Insert Into Memory (vectorStorePGVector, mode=insert)
  │     └── Gemini Embeddings Ingest (sub-node) — gemini-embedding-001, 768 dims
  └── Respond Memory — { status: 'ok', memorise: true }
     ↓
memory_writer reçoit { status: 'ok' }
     ↓
AI Agent confirme en HTML à l'utilisateur
     ↓
[Session suivante]
Utilisateur → "Qu'est-ce que tu sais sur mes préférences ?"
     ↓
Tool: finance_rag → recherche sémantique dans finance_vectors
  → retrouve "[CFO-MEMORY | PREFERENCE | ...] Je préfère les Airbnb..."
     ↓
AI Agent synthétise et répond
```

## §8 — Limites connues et évolutions futures

| Limitation | Impact | Solution future envisageable |
|---|---|---|
| Pas de dédoublonnage | Une même info mémorisée 2x crée 2 vecteurs | Vérification sémantique avant insert (cosine threshold) |
| Pas d'interface de gestion | Impossible de lister/supprimer les mémoires via le chat | Nouvel outil `memory_list` + `memory_delete` |
| TTL absent | Les mémoires durent indéfiniment | Champ `expires_at` dans metadata + cron de nettoyage |
| Catégorie inferée par l'agent | Peut être mal classifiée | Confirmation de catégorie avant mémorisation |
